# 第1章：OpenHarness 是什么

## 1.1 项目定位与愿景

OpenHarness 是一个**企业级开源 Agent 框架**，由香港大学于 2025 年开源发布。它采用 MIT 许可证，目标是将大语言模型（LLM）包装成能**真正干活的全功能智能体**。

**一句话概括：** Claude Code 的开源平替，且多了飞书、钉钉、企业微信等中国市场全覆盖。

---

### 为什么要关注 OpenHarness？

市场上不缺 Agent 框架（LangChain、AutoGPT、OpenManus……），但 OpenHarness 的独特之处在于：

| 维度 | 其他框架 | OpenHarness |
|------|---------|-------------|
| **完成度** | 多数是半成品，需要大量二次开发 | 开箱即用，114 单测通过 + 6 套 E2E 测试 |
| **渠道集成** | 基本只有 Slack/Discord | **飞书、钉钉、企业微信**全覆盖（中国市场刚需） |
| **工具覆盖** | 零散实现，缺这个少那个 | 覆盖 Claude Code **98%** 的工具能力 |
| **企业就绪** | 缺权限、缺上下文管理 | 多级权限沙箱、上下文压缩、会话恢复全套 |
| **代码量** | 动辄数万行，难以理解 | 核心仅 **1.1 万行**（engine 666 行），源码可读性强 |

## 1.2 技术架构概览

### 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 语言 | **Python 3.10+** | 低门槛，团队易于扩展 |
| 前端/CLI | React TUI（基于 Ink） | 终端内交互，类 claude-code 体验 |
| 协议 | **MCP**（Model Context Protocol） | Anthropic 主导的开放标准 |
| 测试 | pytest | 114 单测通过 + 6 套端到端测试 |
| 许可证 | **MIT** | 商用友好，无 AGPL 传染性 |

## 1.3 代码规模分布（精确源码数据）

通过 `wc` 精确统计：**194 个 Python 文件，26,666 行代码**。

| 模块 | 文件数 | 代码行 | 占比 | 一句话说明 |
|------|--------|--------|------|-----------|
| **channels/impl** | 13 | 5,183 | 19.4% | 多渠道消息适配——项目的护城河 |
| **swarm** | 11 | 4,899 | 18.4% | 多 Agent 协同调度 |
| **tools** | 42 | 2,542 | 9.5% | 43 种工具实现 |
| **ui** | 10 | 2,006 | 7.5% | CLI 终端交互 |
| **api** | 9 | 1,468 | 5.5% | RESTful API |
| **commands** | 2 | 1,456 | 5.5% | 54 个 CLI 命令 |
| **coordinator** | 3 | 1,506 | 5.6% | 多 Agent 编排 |
| **services** | 8 | 1,388 | 5.2% | 通用服务层 |
| **plugins** | 6 | 328 | 1.2% | 插件加载器 |
| **config** | 3 | 362 | 1.4% | 配置加载与校验 |
| **engine** | 6 | **666** | 2.5% | ⭐ 核心循环——整本书的主角 |
| 其他 | 33 | 4,875 | 18.3% | 工具类、异常、常量等 |
| **合计** | **194** | **26,666** | **100%** | |

**关键解读：**

> 🔑 **最大的模块不在 engine，而在 channels（渠道层）**。这说明 OpenHarness 的核心竞争力不是"又一个 Agent loop 实现"，而是**"能把 Agent 送到用户日常使用的所有平台上"**。

> 🔑 **engine 只有 666 行**——意味着核心循环极简，非常容易理解。这是本书第 3 章的重点。

> 🔑 **swarm（18.4%）占比超过 tools（9.5%）**——多 Agent 协调是这个项目的战略重心，不只是一个附加功能。

## 1.4 核心特性清单

| 特性 | 描述 | 企业价值 |
|------|------|---------|
| 🔄 **Agent Loop** | 流式工具调用循环，支持并行工具执行 | 低延迟、高吞吐 |
| 🔧 **43+ Tools** | 文件读写、Shell、Web 搜索、MCP、任务管理…… | 覆盖企业日常操作场景 |
| 🧠 **Memory 记忆** | CLAUDE.md 自动注入 + MEMORY.md 持久化 + 会话恢复 | Agent 有"长期记忆" |
| 🛡️ **Permissions 权限** | 严格/宽松/禁用三级模式，路径规则，命令黑名单 | 零信任安全模型 |
| 📚 **Skills 技能** | 按需加载 `.md` 知识库到 system prompt | 教 Agent 专业知识 |
| 🔌 **Plugins 插件** | 兼容 Anthropic Skills + Claude Code Plugins | 复用生态 |
| 🕊️ **Swarm 集群** | 多 Agent 协调、子 Agent Spawn、结果汇总 | 复杂任务分解执行 |
| ⚙️ **Hooks 钩子** | Pre/Post Tool Use 等生命周期拦截器 | 审计、日志、自定义逻辑 |
| 🕐 **Cron 定时任务** | 内置调度，可设 cron 表达式 | 定时报告、巡检 |
| 💬 **Commands 命令** | 54 个斜杠命令（/help, /commit, /plan……） | 用户交互体验 |
| 🌍 **MCP 协议** | Model Context Protocol 客户端支持 | 对接外部工具生态 |

## 1.5 模型兼容性

| 协议格式 | 支持的提供商 | 价格参考 |
|---------|-------------|---------|
| **Anthropic 原生** | Claude 系列、Moonshot（Kimi） | Kimi 约 ¥0.01/千 token |
| **OpenAI 兼容** | 通义千问（DashScope）、DeepSeek、OpenAI、SiliconFlow、Groq | DeepSeek 免费 tier 可用 |
| **本地模型** | Ollama（支持 1000+ 开源模型） | 零 API 费用，硬件成本 |
| **GitHub Copilot** | OAuth 登录，免 API Key | 企业已有 Copilot 许可即可 |

**中国开发者最关心的三个选择：**

1. **通义千问（Qwen）** — DashScope 平台提供，性价比极高，中文能力顶尖
2. **Moonshot（Kimi）** — 中文长文本处理强者，适合知识库场景
3. **DeepSeek** — 有免费 tier，适合低成本试错
